= Visual editor
:toc: right
:imagesdir: ../images

The *Visual editor* is the default <<../preview#,Preview>> mode for renderable content. It is both a preview of the page as end users will see it _and_ a drag-and-drop surface for building that page: components can be added, configured, rearranged, and removed directly in the editor.

The Visual editor works in tandem with the <<../content-form#,Content Form>> (for the content's data fields) and the <<../context-panel#,Context Panel>> (for component configuration and page-level widgets).

== Selecting a controller or template

The Visual editor is hidden for content that cannot be rendered (folders, shortcuts, unstructured content without a supporting template).

For a *Site* without any installed applications that contain a page controller, the editor is shown but disabled:

image::page-editor-no-application.png[]

If an application is installed but no template was matched automatically, the editor shows a dropdown of available templates and controllers:

image::page-editor-no-controller.png[]

Pages rendered by an automatic template are locked against editing by default. Click the page (or an empty region) and choose *Customize* from the menu to enable editing for this content item.

image::page-editor-lock.png[]

== Working with components

Click a component in the page to select it — the selection is highlighted and its configuration appears in the <<../context-panel#_inspect_tab,Inspect tab>> of the Context Panel. Right-clicking a component opens a menu with the available actions:

* *Select parent* — move the selection up the component tree.
* *Insert* — add another component.
* *Inspect* — jump to the component's configuration.
* *Reset* — revert the component to its default configuration.
* *Remove* — delete the component.
* *Duplicate* — make a copy alongside the original.
* *Save as fragment* — convert the component into a reusable <<#_fragment,Fragment>>.

For a *Fragment* component, the last item is replaced with *Detach from fragment*, and an *Edit* action is added.

image::page-editor-menu.png[]

== Component categories

Enonic XP has five categories of components that can be placed in page regions. They are available from the <<../context-panel#_insert_tab,Insert tab>> of the Components widget.

=== Part

Parts are reusable, configurable components supplied by the site's applications. Parts render custom output — for example a list of articles, a form, or a hero banner.

To add a part: open the Insert tab, drag the part placeholder (puzzle-piece icon) onto a region, then use the dropdown in the placeholder to select which part. Once selected, the placeholder is replaced with the rendered part and its configuration form appears in the Inspect tab.

image::component-types-part-edit.png[]

A single part can be added to many pages — or several times on one page — and each instance is configured independently.

image::component-types-part-insert.png[]

TIP: Applications can restrict a part to specific content types. See the https://developer.enonic.com/docs/xp/stable/cms/schemas#part[XP part descriptor reference] for details.

=== Layout

Layouts are configurable containers that define regions where other components can sit. Their primary purpose is to place components side-by-side — typically as two- or three-column structures. Layouts are supplied by applications and may have configuration options such as column widths.

Drag a Layout placeholder into a region, then pick the specific layout from the dropdown. Its configuration appears in the Inspect tab.

image::component-types-layout-insert.png[]
image::component-types-layout-edit.png[]

NOTE: Layouts cannot be nested inside other layouts. If a layout is hard to click in the rendered page, use the Page Components tree (see below) or right-click a child component and use *Select parent*.

=== Text

The Text component is a formatted text block — it places a rich-text surface directly into a page region. Text components support images, videos, links, and macros, all edited via the <<../rich-text-editor#,Rich Text Editor>>.

Drag a Text component into a region and start typing. If another component is selected first, double-click the text component to return to editing.

image::component-types-text-insert.png[]

=== Image

The Image component inserts an image content item directly into a page region. The image is selected from existing content, and any <<../../content-types#_image_content,crop or focal point>> settings on the underlying image content apply automatically when the page is rendered.

[[_fragment]]
=== Fragment

A *Fragment* is a component saved as its own content item so it can be reused on multiple pages with identical configuration. Editing a fragment updates every page that uses it — unlike the other components, which are configured independently on every page.

Fragments are created by right-clicking any component in the Visual editor and choosing *Save as fragment*. The new fragment opens in its own editor tab, and the original component is replaced with a reference to the fragment.

To place an existing fragment on a page, drag a Fragment placeholder from the Insert tab and pick the desired fragment content from the dropdown.

image::component-types-fragment-insert.png[]

== Page Components tree

When the Content form is collapsed (with the "<" icon in the menu toolbar), the Page Components tree undocks into floating mode:

image::wizard-component-view-floating.png[Page Components floating, 75%]

The floating view can be collapsed with the icon in its header; the tree then docks into the vertical menu toolbar and its icon appears at the bottom. Clicking the icon reopens the view.

== Related widgets

When the Visual editor is available, the <<../context-panel#_components,Components widget>> is also available in the Context Panel and is the primary tool for configuring the selected component. Other built-in widgets (Details, Version History, Dependencies) work the same way here as on non-renderable content.
