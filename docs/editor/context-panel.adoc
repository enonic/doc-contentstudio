= Context Panel
:toc: right
:imagesdir: images

The *Context Panel* is the multi-function tool column on the right-hand side of the editor. It hosts a set of *widgets* — small tools that operate on the currently selected content item. Toggle the panel open and closed with the cog icon on the right of the toolbar.

:imagesdir: ../widgets/images
image::widgets-panel-button.png[Context Panel toggle button, 209]

The panel contains a widget selector dropdown: built-in widgets appear first, followed by any custom widgets contributed by installed applications. The top of the dropdown has a filter field for quickly finding a widget by keyword.

image::widgets-search.png[Widget selector with filter, 206]

== Built-in widgets

=== Components

NOTE: The *Components* widget is only available for renderable content with the <<preview/visual-editor#,Visual editor>> enabled.

The Components widget manipulates the page content and has two tabs.

==== Insert tab

The Insert tab lets you drag and drop components onto the page. Enonic XP has five built-in component categories:

* *Layout* — adds a container that defines regions where other components can sit.
* *Part* — adds a configurable component from one of the site's applications.
* *Text* — adds a formatted text block edited via the <<rich-text-editor#,Rich Text Editor>>. Supports images, videos, links, and macros.
* *Fragment* — adds a reusable fragment (a component saved as its own content item and reused across pages).
* *Image* — inserts an image content item directly as a page component.

image::widgets-components-insert.png[Insert tab, 207]

When you drag a placeholder onto the page, a red circle indicates an area where the component cannot be dropped; a green checkmark and blue target box appear over valid drop locations. Once dropped, the placeholder shows a combo-box where you select the specific part/layout/fragment to use.

:imagesdir: images
image::context-panel_insert.png[Placing a part component, 697]
image::context-panel_insert2.png[Selecting the specific part, 440]

==== Inspect tab

The Inspect tab configures whichever component is currently selected in the Visual editor. The tab header changes to match the selected component's type (*Page*, *Part*, *Layout*, etc.). When nothing is selected, it defaults to *Page*.

:imagesdir: ../widgets/images
image::widgets-components-page.png[Page-level inspection, 207]
image::widgets-components-part.png[Part-level inspection, 207]

For the *Page* itself, the Inspect tab is where you choose the controller or page template. For other components, the tab shows the component's configuration form. Changes to configuration values take effect on save or when you hit the *Apply* button at the bottom of the panel.

=== Details

The Details widget shows properties of the selected content item:

* *Content* — icon, display name and full path.
* *Status* — current content status (_New_, _Published_, _Modified_, etc.).
* *User access* — the effective permissions. The *Edit Permissions* link opens the <<../permissions#,permissions dialog>>.
* *Properties* — metadata such as type, created/modified timestamps and owners.
* *Schedule* — the from/to dates the content is online (only shown for published content with a schedule).
* *Page template* — the template used to render the content, if any.
* *Attachments* — downloadable binary attachments.

image::widgets-details.png[Details widget, 450]

=== Version History

Every action on a content item creates a new version. The Version History widget lists all versions and publish/unpublish events and lets you revert, restore and compare them. The current workflow status appears in the widget header.

image::widgets-version-history.png[Version history, 283]

To revert to a specific version, click its box to expand it and then click *Revert*. The current (active) version cannot be reverted and is therefore not expandable — the same applies to publish/unpublish markers.

NOTE: Some versions cannot be reverted — for example those created by permissions updates, rename, or move operations.

Click the image:widgets-version-compare-icon.png[,24] icon on a version to open the *Compare versions* dialog. Two dropdowns at the top select the pair of versions to compare (the left one is older, the right one newer). By default the dialog shows only differences; toggle *Show entire content* to see the full content JSON.

image::widgets-version-compare-dialog.png[Version compare dialog, 909]

Both dropdowns include aliases at the top of the list (_previous_, _next_, _published_, _current_) so you can compare against those without scrolling.

image::widgets-version-compare-dialog-aliases.png[Version aliases, 456]

The kebab menu on the right of each dropdown has a single option — *Revert* — to revert to that version.

=== Dependencies

The Dependencies widget shows the inbound and outbound references of the selected content item.

* An *inbound* dependency is content that uses the selected item.
* An *outbound* dependency is content that the selected item uses.

image::widgets-dependencies.png[Dependencies widget, 207]

Inbound dependencies appear above the content, outbound below. If there are none of a particular kind, the widget shows `No incoming dependencies` (or equivalent). When there are dependencies, a button shows the total count; clicking it opens a new browser tab listing the dependencies. Clicking a per-type icon opens the same view filtered to that type.

image::widgets-dependencies-filter.png[Dependency filter, 370]

== Optional and custom widgets

The Context Panel is *pluggable*. In addition to the built-ins above, any installed application can contribute a widget that will appear in the widget selector.

:imagesdir: images
Widgets shipped via <<../content-studio-plus#,Content Studio+>>:

* <<../layers#layers_panel,Layers>> — shows where the content item exists across every layer in the system.
* <<../variants#,Variants>> — manages A/B variants of the selected content item.

Third-party and custom widgets appear below the built-ins in the dropdown. The interface for contributing a widget to the Context Panel is `contentstudio.contextpanel`; see the https://developer.enonic.com/docs/xp/stable/admin/widgets[XP admin widgets reference] for implementation details.
